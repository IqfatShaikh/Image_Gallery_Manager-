1. PROJECT TITLE

Image Gallery Manager


================================================================================
2. PROJECT DESCRIPTION
================================================================================

The Image Gallery Manager is a modern web application designed to help users
manage and organize image galleries efficiently. The application allows users
to add, view, edit, and delete images using external URLs without requiring
direct file uploads. 

The primary purpose of this application is to provide a user-friendly platform
for creating and managing image collections. This is particularly useful for:

- Portfolio websites showcasing creative work
- E-commerce platforms displaying product images
- Content creators managing visual collections
- Archives and documentation systems storing reference images
- Blogs and news websites organizing visual content

The application combines a sleek user interface with powerful backend
functionality to ensure smooth image management operations.

5. FUNCTIONAL REQUIREMENTS
================================================================================

The application must support the following features:

1. Add Image
   - Users can submit a form with image URL, title, and description
   - The system validates and stores the image information in the database
   - Users receive confirmation upon successful submission

2. View All Images
   - Users can see a grid or list of all stored images
   - Images display with thumbnail, title, and description
   - Users can easily navigate through the gallery

3. View Image Details
   - Users can click on an image to see full details
   - Displays complete information including image, title, description, and dates
   - Shows metadata about when the image was added

4. Edit Image Details
   - Users can access an edit form for any image
   - Allows updating title, description, and URL
   - Changes are saved to the database immediately

5. Delete Image
   - Users can remove images from the gallery
   - Deletion is confirmed before removing data
   - Deleted images are permanently removed from the database

7. API DESIGN
================================================================================

The application implements RESTful APIs for image management operations:

API Name: Add Image
Method: POST
Endpoint: /api/images
Description: Creates a new image entry in the database with provided details
Parameters: title, imageUrl, description
Response: Created image object with ID and timestamp

---

API Name: Get All Images
Method: GET
Endpoint: /api/images
Description: Retrieves a list of all images in the gallery
Parameters: Optional pagination parameters
Response: Array of image objects

---

API Name: Get Image by ID
Method: GET
Endpoint: /api/images/{id}
Description: Retrieves detailed information about a specific image
Parameters: Image ID in URL path
Response: Single image object with complete details

---

API Name: Update Image
Method: PUT
Endpoint: /api/images/{id}
Description: Updates image information such as title or description
Parameters: Image ID in URL path, updated fields in request body
Response: Updated image object

---

API Name: Delete Image
Method: DELETE
Endpoint: /api/images/{id}
Description: Permanently removes an image from the database
Parameters: Image ID in URL path
Response: Confirmation message or deleted image object

8. DATABASE SCHEMA
================================================================================

The application uses a simple yet effective database schema for image storage:

Table Name: images

Columns:

- id (Integer, Primary Key, Auto-increment)
  Purpose: Unique identifier for each image entry
  Type: Integer
  Constraints: Primary key, unique, not null

- title (String, Required)
  Purpose: Display name of the image
  Type: String (255 characters)
  Constraints: Not null, indexed for searching

- imageUrl (String, Required)
  Purpose: External URL pointing to the image file
  Type: String (500 characters)
  Constraints: Not null, must be valid URL

- description (Text, Optional)
  Purpose: Detailed explanation or metadata about the image
  Type: Text
  Constraints: Nullable, allows longer content

- createdAt (Timestamp, System-Generated)
  Purpose: Records when the image was added to the gallery
  Type: DateTime
  Constraints: Auto-generated, not null

- updatedAt (Timestamp, System-Generated)
  Purpose: Tracks the last modification time of the image record
  Type: DateTime
  Constraints: Auto-updated on changes

The project is organized following Next.js best practices:

Root Directory: /
  Contains configuration files and documentation

│
├── /src
│   │
│   ├── /app
│   │   Purpose: Next.js App Router directory containing page components
│   │   Contents:
│   │   - layout.tsx: Root layout wrapper for all pages
│   │   - page.tsx: Home page displaying the image gallery
│   │   - globals.css: Global styles applied across the application
│   │   
│   │   Subdirectories:
│   │   - /add: Page for adding new images
│   │   - /edit/[id]: Dynamic page for editing specific images
│   │   - /image/[id]: Dynamic page for viewing image details
│   │
│   ├── /components
│   │   Purpose: Reusable React components used throughout the application
│   │   Contents:
│   │   - ImageCard.tsx: Component displaying individual image tiles

│   │   - ToastProvider.tsx: Toast notification system component
│   │
│   ├── /lib
│   │   Purpose: Utility functions and helper modules
│   │   Contents:
│   │   - prisma.ts: Database client instance and utilities

│   └── actions.ts: Next.js server actions for API operations

├── Configuration Files

│   - next.config.ts: Next.js configuration
│   - eslint.config.mjs: Code linting rules
